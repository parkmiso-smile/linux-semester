# 13주차 감상문

이번 한 학기 동안 진행한 '리눅스 한 학기 살기' 프로젝트를 통해서 직접 도구를 설치하고 보안 관련 실습을 설계하며 실제 CTF 문제를 만들어 보았다.

리눅스는 윈도우와 달리 사용자가 파일 시스템 구조(`/home`, `/etc`, `/var/www/html` 등)를 직접 이해해야 하고, 서비스의 실행·중지·포트 바인딩을 명령어로 제어해야 한다. 처음에는 이러한 구조가 생소했지만, 매주 새로운 패키지를 설치하며 `apt`, `systemctl`, `ss`, `chmod`, `nano`, `scp` 같은 명령어들이 자연스럽게 익숙해졌다. 특히 서비스 기반 실습(vsftpd, OpenSSH, Bind9 등)을 하며 "리눅스는 서비스가 어떻게 동작하고 어떤 설정 파일에서 제어되는지"를 직접 체험할 수 있었다.

"각 주차마다 하나의 프로그램을 직접 설치하고, 그 도구를 활용해 CTF 문제를 설계한다"는 목표로, 이번 프로젝트는 리눅스와 보안을 동시에 익힐 수 있는 흥미로운 경험이었다.

- **Vim**: 공백 기반 스테가노그래피 문제 제작
- **curl**: 헤더 변조를 통한 웹 해킹 우회 문제 제작
- **tar/gzip**: 깊은 디렉터리 구조 속 숨겨진 증거 파일을 찾는 포렌식 시나리오 구성
- **vsftpd**: FTP 로그인 구조와 숨김 파일 노출 위험성 실습
- **Netcat/Nmap**: 포트 스캔 및 서버 플래그 획득 문제 제작
- **Bind9**: DNS zone 파일 변조 공격 시뮬레이션 ("도메인 이름 하나만 바뀌어도 전체 서비스가 공격자 의도대로 이동할 수 있다")
- **FFmpeg**: 오디오 인코딩·암호화·역재생 CTF 문제 제작
- **Hydra/OpenSSH**: 실제 비밀번호 무차별 대입 공격 재현
- **SET**: 피싱 페이지 구성 및 동작 원리 확인

이번 프로젝트를 통해 단순히 보안 도구를 사용하는 수준을 넘어, 리눅스 시스템을 직접 구성하고 통제하면서 보안의 원리에 대해서도 학습할 수 있었다. 무엇보다도 실습을 통해 배운 보안 개념들이 모두 리눅스의 파일 시스템 구조, 프로세스 관리, 권한 모델 위에서 작동한다는 사실을 알 수 있었다.
